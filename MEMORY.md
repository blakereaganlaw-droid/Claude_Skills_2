## Lessons Learned & Avoidance Rules
- RULE: Never chain an evidence-gathering command with a destructive command (especially
  `git push --force*`) in one shell call — the destructive step runs before the evidence is
  read. Verify in one step, act in the next. (origin: 2026-07-18 force-push incident, waves
  4–9 briefly displaced, restored via merge `be2c8af`)
- RULE: Before ANY push to the shared branch, fetch and inspect the remote tip separately —
  other agents push to it; "only already-merged history" must be proven by a content diff
  against main, then acted on in a separate command. (origin: same incident)
- RULE: After a PR from the working branch merges, restart the branch from `origin/main`
  before new work — never stack on merged history. (established practice, works cleanly)
- LESSON: `/plugin` commands exist only in local Claude Code (CLI/desktop/IDE), not in cloud
  sandbox sessions; in the cloud use the `claude plugin` CLI, which works. (observed 2026-07-18)
- LESSON: With ~100 installed skills, the skill-listing context budget trims least-used
  descriptions to name-only; direct `/plugin:skill` invocation still works, and newly installed
  plugins appear in the listing only at the next refresh. (observed 2026-07-18)
- LESSON: The GitHub App integration cannot create repositories (403 "Resource not accessible
  by integration") — the user creates the empty repo manually, then `add_repo` brings it into
  session scope for pushing. (observed creating BSL_MATCHING_ENGINE, 2026-07-18)
- RULE: Never pass reference-bearing Oracle/bank exports through pandas or an Excel resave —
  float coercion turns `0006789599` into `6789599.0` and destroys join keys; the recon engines
  ban pandas entirely for this reason. (source: OG_Recon README, engine-enforced)
