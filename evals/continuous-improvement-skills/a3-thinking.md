# Evals — continuous-improvement-skills:a3-thinking

## 1. Positive trigger (should load the skill)
> "I need to propose a fix for our slow invoice-approval process to the controller and get buy-in. Can
> you help me put it on a one-page A3 with the PDCA story?"

Expected: skill loads; walks the A3 boxes in order — background, current condition with data, measurable
target, verified root cause, countermeasures that trace to the cause, an owned implementation plan, and a
follow-up/check with a date; maps them to PDCA; keeps it to one page; frames the A3 as a consensus-building
thinking tool drafted *with* stakeholders.

## 2. Near-miss (should NOT load this skill)
> "Help me actually run the 5 Whys and build a fishbone to pin down why invoices get approved late."

Expected: this is the deep cause-analysis work that feeds an A3's analysis box, handled by
`continuous-improvement-skills:root-cause-analysis`. The A3 is the one-page story around it. If a3-thinking
loads instead, tighten the cross-links.

## 3. Quality rubric
A good response:
- **Does the task:** produces a single-page A3 with background, data-backed current condition, measurable
  target, verified root cause, cause-traceable countermeasures, a dated implementation plan, and a
  follow-up/check — mapped to Plan-Do-Check-Act.
- **Teaches:** explains *why* the A3 is a thinking and alignment tool (the value is the dialogue it forces),
  why the one-page limit is a discipline, and why every countermeasure must trace to a cause.
- **Grounded:** insists on data in the current-condition box, keeps it to one page, and treats Check/Act as
  boxes filled in later with real results — a learning loop, not a frozen proposal.
