# Evals — continuous-improvement-skills:dmaic-problem-solving

## 1. Positive trigger (should load the skill)
> "I want to run a proper Six Sigma project to reduce our reconciliation break rate. I need a charter,
> a baseline, a way to prove the cause, and a control plan so it doesn't creep back. Walk me through
> DMAIC."

Expected: skill loads; works Define → Measure → Analyze → Improve → Control with tollgates; writes a
charter with measurable CTQs; builds a trustworthy baseline (with a measurement-system check); verifies
the cause with data before improving; pilots before rollout; ends with standard work plus SPC/monitoring
and an owner.

## 2. Near-miss (should NOT load this skill)
> "Something went wrong with yesterday's payment run — can you quickly help me 5-Whys it and find the
> cause?"

Expected: this is a fast one-off cause hunt with no baseline or control plan, which
`continuous-improvement-skills:root-cause-analysis` handles. DMAIC is heavier machinery than the ask
warrants. If dmaic-problem-solving loads instead, tighten the cross-links.

## 3. Quality rubric
A good response:
- **Does the task:** produces a charter with measurable goal/CTQs, a baseline with a measurement-system
  sanity check, a data-verified cause, a piloted solution measured against baseline, and a control plan
  with monitoring and an owner.
- **Teaches:** explains *why* the sequence forces measuring and confirming cause before changing
  anything — the thing that separates DMAIC from guess-and-check — and why Control is where gains are
  usually lost.
- **Rigorous:** holds the tollgate discipline, tests the cause rather than asserting it, and distinguishes
  a real shift from normal variation.
