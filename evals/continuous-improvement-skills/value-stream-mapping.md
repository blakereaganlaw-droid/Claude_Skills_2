# Evals — continuous-improvement-skills:value-stream-mapping

## 1. Positive trigger (should load the skill)
> "Our invoice-to-cash process takes about three weeks and nobody knows why. Can you help me map the
> whole value stream, current and future state, and figure out where all the time goes?"

Expected: skill loads; scopes with SIPOC; walks the process capturing cycle time, wait time, and
%C&A per step; builds the timeline ladder and computes process cycle efficiency; tags the 8 wastes;
designs a future state with an owned action plan. Shows most lead time is waiting, not working.

## 2. Near-miss (should NOT load this skill)
> "Invoices keep getting posted to the wrong GL account. Can you help me find the root cause?"

Expected: this is diagnosing one recurring defect, not mapping a whole process. The
`continuous-improvement-skills:root-cause-analysis` skill should handle it (5 Whys / fishbone). If
value-stream-mapping loads instead, tighten the description and cross-links.

## 3. Quality rubric
A good response:
- **Does the task:** scopes with SIPOC, maps steps with data boxes (CT, %C&A) and inter-step waits,
  computes lead time vs. process time and flow efficiency, identifies the 8 wastes, and produces a
  future-state map plus a dated action plan with targets.
- **Teaches:** explains *why* lead time is dominated by queues (Little's Law) so the fix is smoothing
  flow and building quality in at the source, not making steps faster.
- **Grounded:** treats the current-state map as a measured baseline, not the official procedure, and
  the future state as a hypothesis to test — not a one-off diagram.
