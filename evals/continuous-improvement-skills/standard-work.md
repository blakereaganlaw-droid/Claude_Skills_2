# Evals — continuous-improvement-skills:standard-work

## 1. Positive trigger (should load the skill)
> "Every analyst does our daily cash reconciliation differently and the results vary. Can you help me
> write standard work / an SOP for it so it's done the same way every time?"

Expected: skill loads; scopes one repeatable task with its trigger; documents the current best method
observed at the gemba; records sequence, cycle time (and takt/volume context), and standard inputs;
captures key points *with reasons*; adds visual management; covers training verification and a review
cadence; frames the SOP as a living baseline for improvement.

## 2. Near-miss (should NOT load this skill)
> "Help me build the month-end close calendar — sequence all the tasks across the team with owners and
> business-day offsets."

Expected: this is sequencing a whole multi-task close, which `accounting-skills:month-end-close` handles;
standard work documents *one* repeatable task's method. If standard-work loads instead, tighten the
cross-links. (Standardizing one close task's method, though, is a legitimate standard-work use.)

## 3. Quality rubric
A good response:
- **Does the task:** produces standard work for one task with sequence, timing (takt/cycle context),
  standard inputs, key points *and their reasons*, visual management, training verification, and a
  review cadence.
- **Teaches:** explains *why* you can't improve an unstable process and that a standard is the baseline
  every kaizen measures against — a living document, not a binder.
- **Grounded:** documents the real current best method (not an aspirational one) and builds in a path to
  surface deviations and better ideas rather than freezing the process.
